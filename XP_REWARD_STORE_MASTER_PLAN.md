# LumenPortal — XP, Coins & Experience Store Master Plan

> תאריך יצירה: 2026-04-29  
> סטטוס: תוכנית ביצוע חדשה, Priority 2 אחרי Finish Line 1  
> מטרה: להפוך כל תשובה, הישג והתקדמות למערכת העצמה מתמשכת עם 100 רמות XP, מטבעות חוויה, חנות פרסים, ומוזיאון נעול כפרס למידה.

---

## 1. עקרון יסוד

המערכת מפרידה בין שלושה דברים:

1. **שליטה לימודית** — ציון אמיתי במושג. אי אפשר לקנות אותו.
2. **XP** — מדד התקדמות גלובלי שמצטבר מכל פעילות למידה.
3. **Coins** — מטבע חוויה שמתקבל יחד עם XP ומשמש לפתיחת חוויות, מוזיאון, עיצובים ואתגרים.

כלל ברזל: **חנות לא פותחת ציון, לא עוקפת שאלות, לא נותנת "מאסטר" ולא מסתירה חומר חובה למבחן.** היא מוכרת חוויה, מוטיבציה וקוסמטיקה.

## 3. מודל 100 רמות XP

### 3.1 רמות גלובליות

הרמות הגלובליות יושבות מעל 7 רמות המושג הקיימות. תלמיד יכול להיות ברמה גלובלית גבוהה ועדיין להיות חלש במושג מסוים.

| טווח | שם דרגה | משמעות |
|---:|---|---|
| 1-10 | סבתא מתכנתת | התחלה, מושגים בסיסיים, הרבה חיזוק |
| 11-25 | תלמיד כיתה א' | מזהה מושגים ופותר שאלות קלות |
| 26-40 | תלמיד חטיבה | מבין קשרים בין מושגים וטעויות נפוצות |
| 41-55 | תלמיד תיכון | פותר קוד בינוני ומתקן באגים פשוטים |
| 56-70 | ג'וניור מתכנת | בונה פיצ'רים קטנים ומבין זרימת נתונים |
| 71-85 | מפתח עצמאי | מחבר frontend/backend/db/auth באופן מעשי |
| 86-99 | מאסטר קורס | שולט ברוב הקורס תחת שאלות עומק וקוד |
| 100 | ציון 100 | מאסטר מלא בכל מושגי הפורטל + פתרון קטעי קוד לכל המושגים הרלוונטיים |

### 3.2 עקומת התקדמות לא לינארית

התקדמות ראשונה צריכה להיות מהירה. ככל שמתקרבים לרמות גבוהות, העלייה נהיית קשה יותר.

נוסחת יעד מוצעת:

```text
xpRequired(level) = floor(35 * level^1.85 + 15 * level)
```

מאפיינים:
- רמות 1-10 עולות מהר כדי לתת מוטיבציה.
- רמות 40-70 דורשות עקביות ותרגול אמיתי.
- רמות 86-99 דורשות שליטה רחבה, לא רק צבירת שאלות קלות.
- רמה 100 היא לא רק XP: היא שער הוכחה.

### 3.3 שער רמה 100

רמה 100 דורשת את כל התנאים:

1. 100% מושגים באתר במצב `mastered`.
2. לכל מושג: פתרון נכון של שאלת האתגר הגבוהה ביותר הזמינה.
3. לכל מושג קוד: השלמת/Trace/Debug/Build רלוונטי שעבר.
4. אין מושגים מסומנים כחלשים ללא תיקון לאחר מכן.
5. מבחן מדומה מלא בציון 95+ לפחות פעם אחת.
6. `validate:strict`, `qa:questions:strict`, build ו-smoke ירוקים.

## 5. Coins Economy

### 5.1 מטבע

שם מוצע: **Lumen Coins**.

כללי שמירה:
- נשמר מקומית לפי תלמיד.
- מיוצא/מיובא עם progress.
- לא קשור לכסף אמיתי בשלב זה.
- אי אפשר להעביר בין תלמידים.
- אי אפשר לקנות ציון או לדלג על שאלות.

### 5.2 סוגי רכישות

| קטגוריה | דוגמאות | מחיר מוצע |
|---|---|---:|
| כרטיס מוזיאון | כניסה לאגף שפות, אגף חשמל, אגף React, אגף AI | 20-120 coins |
| חדרי חוויה | Escape Room Debug, Boss Fight של Async, API Mystery | 80-250 coins |
| עיצוב וקוסמטיקה | Theme, badge, avatar title, צבע XP bar | 25-180 coins |
| כלי לימוד חווייתי | רמז ויזואלי, replay אנימציה, מסלול ביקור מודרך | 10-60 coins |
| אתגרי פרימיום | מבחן בוס, סימולציית ראיון, Capstone challenge | 150-400 coins |
| אספנות | חותמות מוזיאון, קלפי מושג, trophies | 15-90 coins |

## 6. נעילת מוזיאון

### 6.1 מה ננעל

המוזיאון הופך ל-Hub חווייתי עם אזורי כרטיסים:

1. **שער פתוח חינם:** מבוא קצר שמסביר למה המוזיאון קיים.
2. **אגפי עומק נעולים:** שפות, חשמל, React, Node, AI, Debug, Product.
3. **חדרים מיוחדים:** Time Machine, שושלות שפות, תיאטרון השוואות, חדר טעויות.
4. **סיורי VIP:** מסלול לפני מבחן, מסלול Full Stack end-to-end, מסלול Debug.

### 6.2 מה לא ננעל

לא נועלים:
- חומר חובה למבחן SVCollege.
- הסברים שנדרשים כדי לענות על שאלה.
- דרישות קדם לשאלה.
- תיקון טעות אחרי כישלון.
- כל תוכן שמונע מהתלמיד להגיע לציון 100.

אם אזור מוזיאון מסביר מושג חובה, תהיה גרסת לימוד חינמית קצרה בכרטיס המושג. המוזיאון מוכר את החוויה המורחבת, לא את הידע החובה.

## 7. חנות חוויות

### 7.1 אזורים שניתן לפתוח

| אזור | חוויה | למה תלמיד ירצה לקנות |
|---|---|---|

| Debug Arena | סדרת תקלות עם דרמות ופתרון | מחזק תיקון באגים |
| Boss Battles | קרב נגד נושא קשה כמו Async/Auth/React State | מוכיח עומק |
| Code Cinema | סרטוני מושג ו-replay של תהליך קוד | למידה חזותית |
| Time Machine | התפתחות שפות וטכנולוגיות | חיבור היסטורי |

| Theme Shop | רקעים, צבעי sidebar, badge titles | תחושת בעלות |
| Mentor Cards | קלפי טיפים של "מנטור" לפי מושג | רמזים ללא הורדת רמת קושי |

### 7.2 מחיר דינמי

מחיר יכול להיות תלוי ב:
- רמת תלמיד גלובלית.
- האם זה אזור חובה או בונוס.
- נדירות החוויה.
- האם התלמיד פתח כבר אזורים דומים.

המחיר חייב להיות דטרמיניסטי:

```text
price = basePrice + difficultyWeight + rarityWeight - masteryDiscount
```

אין שימוש ב-random.

## 9. Data Model

```js
learnerEconomy = {
  xp: 1240,
  coins: 86,
  level: 18,
  lifetimeXp: 1240,
  lifetimeCoinsEarned: 110,
  purchases: {
    "museum.react-evolution": {
      purchasedAt: "2026-04-29T00:00:00.000Z",
      price: 90
    }
  },
  rewardLog: [
    {
      id: "stable-hash",
      at: "2026-04-29T00:00:00.000Z",
      source: "question",
      conceptKey: "lesson21::React",
      xp: 14,
      coins: 4
    }
  ]
}
```

כל ID צריך להיות hash יציב מתוכן/זמן פעולה ידוע, לא randomness.

## 10. Anti-Cheating Rules

1. שאלה שנענתה בעבר לא נותנת XP נוסף.
2. שאלה קלה לא יכולה לקדם לרמות גלובליות גבוהות אחרי תקרה מסוימת.
3. XP גבוה ניתן רק כאשר יש `conceptKey`, `questionId`, `difficulty`, ו-answer result.
4. רמה 100 חסומה עד שכל שערי מאסטר אמיתיים ירוקים.
5. רכישת חוויה לא משפיעה על score.
6. הפסד במסלול מסוכן יכול להוריד coins/XP זמינים, אבל לא למחוק evidence לימודי.
7. כל שינוי ב-XP/Coins נרשם ב-rewardLog.

## 11. שלבי ביצוע

### Phase A — Economy Core

- להחליף `awardXP(amount)` ב-`awardLearningReward({ xp, coins, source, conceptKey, questionId })`.
- להוסיף `getLearnerLevelFromXP(xp)` עם עקומה לא לינארית.
- להוסיף `coins` מקומיים לפי תלמיד.
- להוסיף rewardLog מקוצר.
- להוסיף בדיקות: XP לא שלילי, coins לא שליליים, אין reward כפול לאותה שאלה.

### Phase B — 100 Levels UI

- לעדכן widget עליון לרמה + XP + coins.
- להוסיף progress bar לרמה הבאה.
- להוסיף panel עם פירוט רמות.
- להוסיף badges לפי bands.
- להוסיף gate ברור לרמה 100.

### Phase C — Store MVP

- טאב `🛒 חנות`.
- קטלוג מוצרים דטרמיניסטי.
- רכישה מקומית עם coins.
- רשימת רכישות שלי.
- בדיקה שלא ניתן לקנות אם אין coins.

### Phase D — Museum Tickets

- לנעול אגפי מוזיאון חווייתיים.
- להשאיר חומר חובה פתוח בכרטיסי לימוד.
- להוסיף כרטיסי כניסה לפי אגף.
- להוסיף CTA: "פתיחה עם coins" או "פתיחה דרך שליטה במושג".
- להוסיף smoke למוזיאון נעול/פתוח.

### Phase E — Experience Rewards

- Debug Arena locked rooms.
- Boss Battles לפי נושא.
- Code Cinema / replay clips.
- Secret Labs.
- Theme/cosmetic shop.

### Phase F — Balancing

- לבדוק כמה coins תלמיד מקבל ביום לימוד רגיל.
- לוודא שמוזיאון בסיסי נפתח מהר, ואזורים מיוחדים דורשים מאמץ.
- להוסיף dashboard לכלכלה: earned/spent/unlocked.
- להוסיף tuning table בלי לשנות קוד.
