# תרגילי לחץ זמן — 10 שאלות ב-5 דקות

**מטרה:** לאמן מהירות החלטה תחת לחץ.

**הוראות:**
1. הפעל טיימר 5 דקות.
2. ענה על כל 10 השאלות.
3. אם נתקעת — נחש ועבור הלאה.
4. בדוק את עצמך עם מפתח התשובות.
5. עשה תרגיל אחר ביום — לא יותר. אחרת מתחילים לטעות בגלל עייפות.

---

## תרגיל #1 — 10 שאלות / 5 דקות

**זמן מומלץ:** 30 שניות לשאלה

### 1. מה נכון לתאר את AI לפי השיעור?

A. מודל שמחזיר טקסט/קוד בהסתמך על סטטיסטיקה של ההקשר, לא על הבנה מלאה של אמת
B. מסד נתונים שמחזיק עובדות אמתיות ומעדכן אותן בזמן אמת תמיד
C. מערכת שמחליפה תמיד את שלבי בדיקות ה-production
D. אלגוריתם שמחשב רק מספרים ואסור לכתוב בו טקסט

### 2. מה היתרון והסיכון של שימוש ב-`temperature` בקשת LLM?

A. משפר עקביות ותשובה יציבה יותר, אבל עלול להקטין יצירתיות בתרחישים מסוימים
B. מבטל צורך בכל בדיקות QA
C. מחליף בכלל קונטקסט ולפעמים מכבה את המודל
D. גורם תמיד לעבוד רק עם מידע ציבורי עדכני

### 3. למה צריך להבחין בין `system` ל-`user` בשיחה מול מודל?

A. כי ה-system מגדיר תפקיד והתנהגות כללית, וה-user מגדיר את המשימה בפועל
B. כי רק system יכול לשלוח שגיאות
C. כי user מיועד לטקסטים רק לאנגלית ואילו system רק בעברית
D. כי כזה מבנה מונע שימוש ב-API keys

### 4. מתי נכון להשתמש ב-Cursor Cmd+K?

A. כשיודעים איזה שינוי קטן רוצים לערוך במקום בקובץ הנוכחי
B. כשצריך למחוק את כל ה-repo
C. כשצריך להחליף את Git
D. כשצריך לפרוס production בלי בדיקות

### 5. מה התפקיד של קובץ .cursorrules בפרויקט?

A. לתת ל-Cursor כללי פרויקט קבועים כמו stack, איסורי תלויות וסגנון בדיקות
B. להחליף את package.json
C. לשמור סיסמאות API
D. להריץ build אוטומטי בכל keypress

### 6. מה הסיכון המקצועי בקבלת הצעת Cursor בלי לקרוא diff?

A. ה-AI יכול להכניס שינוי לא קשור, שבירת contract או תלות חדשה בלי שהבנת
B. הקוד הופך למהיר יותר רק בגלל שה-AI כתב אותו
C. הדפדפן מפסיק להציג HTML בגלל כל הצעת autocomplete
D. Git מפסיק לשמור היסטוריה

### 7. מה חסר ב-prompt חלש כמו 'תתקן את זה'?

A. מטרה, context, מגבלות וקריטריון קבלה שאפשר לבדוק
B. ניסוח רגשי יותר בלי לציין קובץ, מגבלה או בדיקה
C. בחירת צבע Tailwind בלי קשר לבאג או לדרישת המשימה
D. מזהה מספרי שרירותי שאינו מוסיף הקשר או קריטריון קבלה

### 8. איזה prompt הכי מתאים לבקשת refactor בטוחה?

A. שנה רק את src/auth.js, אל תוסיף תלויות, שמור API קיים, והריץ npm test
B. שפר את כל הפרויקט בלי לציין קובץ, בדיקה או contract
C. כתוב קוד יפה בלי להגדיר התנהגות צפויה או בדיקה
D. תן לסוכן לבחור scope בלי reviewer ובלי גבולות תלות

### 9. למה כדאי לכלול דוגמאות נגדיות או constraints ב-prompt?

A. כדי למנוע מהמודל לבחור פתרון שנשמע נכון אבל מפר כלל חשוב בפרויקט
B. כדי להאריך את התשובה בלי סיבה
C. כדי לגרום למודל לדלג על בדיקות
D. כדי להחליף manual review

### 10. מה AI Code Review צריך לבדוק מעבר ל-'הקוד נראה יפה'?

A. לוגיקה, אבטחה, contract, בדיקות, edge cases ושינוי לא קשור
B. רק מספר השורות בקובץ
C. רק צבעי Tailwind
D. רק אם יש emoji בתגובה
