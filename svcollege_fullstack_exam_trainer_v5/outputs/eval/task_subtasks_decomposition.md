# פירוק מעמיק למשימות משנה

## client_form_inputs (appearances: 33)
1. לתכנן מבנה טופס לפי דרישה (fields + types).
2. לבנות controlled inputs לכל שדה.
3. להגדיר select/options כולל default תקין.
4. לקשור כפתור submit ל-handler יחיד.
5. לאפס/לעדכן state לאחר submit תקין.
6. למנוע submit כפול בזמן בקשה async.

## client_validation_rules (appearances: 31)

2. להפריד ולידציה סינכרונית מול ולידציה תלוית שרת.
3. להחזיר הודעות שגיאה ספציפיות לכל שדה.
4. לחסום submit כשיש שגיאה אחת לפחות.
5. לבדוק edge cases: ריק, תו מיוחד, גבולות.
6. להבטיח עקביות בין ולידציה client ל-server.
