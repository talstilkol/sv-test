4. Checklist עמוד React

חובה

  • Route מוגדר.
  • כל input controlled.
  • validation לפני fetch.
  • error מתחת לשדה או alert לפי הדרישה.
  • navigate אחרי הצלחה אם צריך.
  • reset אחרי submit אם צריך.

נפילות יקרות

  • שדה לא נבדק.
  • כפתור פעיל למרות קלט לא תקין.
  • layout לא דומה לתמונה.
  • שגיאות console.
  • שימוש בנתונים מומצאים במקום נתוני שאלה/קלט משתמש.

7. תבנית TS 10 נקודות

type BaseItem = {
  id: string;
  title: string;
};

type ActiveItem = BaseItem & {
  kind: "active";
  score: number;
};

type ArchivedItem = BaseItem & {
  kind: "archived";
  reason: string;
};

type Item = ActiveItem | ArchivedItem;

function describeItem(item: Item): string {
  if (item.kind === "active") {
    return String(item.score);
  }
  return item.reason;
}

המטרה: להראות שאתה יודע ליצור טיפוס, union, ולבצע narrowing לפי שדה.

8. Stop Conditions במבחן

  • אם נתקעת 10 דקות על פרט עיצובי - דלג וחזור בסוף.
  • אם API לא עובד - בדוק server רץ, URL, CORS, JSON, status.
  • אם state לא מתעדכן - בדוק mutation ישיר ו-key.
  • אם validation חסר - עצור ותקן; זה מוריד הרבה נקודות.
  • אם משהו לא כתוב בדרישה - לא בונים אותו בזמן מבחן.
